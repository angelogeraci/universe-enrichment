# Facebook Marketing API Configuration

## Prerequisites

To use the Facebook integration in your application, you need to configure the following environment variable:

```bash
# Facebook Marketing API - ONLY REQUIRED VARIABLE
FACEBOOK_ACCESS_TOKEN="your-token-here"
```

**✅ IMPORTANT:** You no longer need `FACEBOOK_AD_ACCOUNT_ID`. The new system only uses the access token.

## Getting Facebook Access Token

### 1. Create a Facebook Application

1. Go to [Facebook Developers](https://developers.facebook.com/)
2. Create a new application
3. Select "Business" as usage type
4. Add the "Marketing API" product

### 2. Generate an Access Token

#### Option A: Via Graph API Explorer (Recommended for Testing)

1. Go to [Graph API Explorer](https://developers.facebook.com/tools/explorer/)
2. Select your application
3. Click "Generate Access Token"
4. Grant permissions:
   - `public_profile` (basic permission)
   - `pages_read_engagement` (optional)
5. Copy the generated token

#### Option B: Via Your Application (Production)

1. Implement OAuth 2.0 flow
2. Request appropriate permissions
3. Retrieve long-lived access token

### 3. Verify Your Token

Test your token with this request:
```bash
curl "https://graph.facebook.com/v18.0/me?access_token=YOUR_TOKEN"
```

# Facebook Ads API Configuration

## Overview

The enrichment system uses Facebook Marketing API to find relevant advertising interests based on criteria generated by OpenAI. This approach enables creating ultra-targeted advertising campaigns.

## Configuration

### 1. Facebook API Keys
Create a `.env` file at project root with:
```
FACEBOOK_ACCESS_TOKEN=your_facebook_access_token_here
```

### 2. Access Token Generation
1. Go to [Facebook for Developers](https://developers.facebook.com/)
2. Create an application
3. Add "Marketing API" product
4. Generate access token with `ads_read` permissions

## System Operation

### Architecture
- **Search**: Facebook Graph API searches for advertising interests
- **Scoring**: Advanced algorithm to evaluate relevance
- **Selection**: Best match automatically identified
- **Interface**: Display with dropdown to explore all suggestions

## 🚀 Scoring System v2.0 Improvements

### Problems Solved

#### 1. Ford Ka vs Ford Motor Company Issue
**Problem**: Old algorithm sometimes selected specific models (e.g., "Ford Ka") instead of main brands (e.g., "Ford Motor Company").

**Solution**: 
- **New brand score (30% of final score)**: Prioritizes entities with brand indicators
- **Model pattern detection**: Identifies and penalizes specific models
- **Brand indicator bonus**: +0.5 for "Company", "Motor", "Corp", etc.
- **Model penalty**: -0.4 for "Brand Model" patterns (e.g., "Ford Ka")

#### 2. Relevance Threshold System
**New**: Automatic suggestion classifications:

| Level | Score | Description | Action |
|-------|-------|-------------|---------|
| 🟢 Very High | ≥80% | Perfect match | Auto-selection recommended |
| 🟡 High | ≥60% | Very good match | Selection recommended |
| 🟠 Medium | ≥30% | Acceptable match | Review recommended |
| 🔴 Low | 15-29% | Doubtful match | **NOT RELEVANT** |
| ⚫ Non-relevant | <15% | No match | **REJECTED** |

### New Score Weighting

```typescript
const finalScore = (
  textualSimilarity * 0.25 +    // 25% - Textual similarity (reduced)
  contextualScore * 0.25 +      // 25% - Contextual relevance  
  brandScore * 0.30 +           // 30% - NEW: Brand score (high priority)
  audienceScore * 0.15 +        // 15% - Audience size (reduced)
  interestTypeScore * 0.05      // 5% - Interest type (reduced)
)
```
